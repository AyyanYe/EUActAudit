# Security Policy

Thank you for taking the time to review the security policy for **EUActAudit (AuditGenius)**. Security and privacy are critical priorities for this project, especially given its role in compliance and auditing.

## Supported Versions

Currently, as the project is in its initial open-source phase, only the latest release and the main branch are supported with active security updates.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| `1.0.x` | :white_check_mark: |
| `< 1.0` | :x:                |

## Reporting a Vulnerability

We take all security vulnerabilities seriously. If you discover a security vulnerability within this project, please do **not** disclose it publicly by creating a public GitHub Issue.

Instead, please report it privately using one of the following methods:

1. **Email:** Send a detailed report to [ayyanahmed@outlook.com](mailto:ayyanahmed@outlook.com).
2. **LinkedIn:** Reach out to me directly via [LinkedIn Direct Message](https://linkedin.com/in/ayyan-ahmed).

### What to expect
* **Acknowledgment:** I will do my best to acknowledge receipt of your vulnerability report within 48 hours.
* **Triage & Update:** You can expect an update on the status of the vulnerability assessment within 5 business days.
* **Resolution:** If the vulnerability is accepted, I will work to patch it as quickly as possible. You will be kept in the loop and, if you wish, credited in the release notes when the fix is deployed.

Please include detailed steps to reproduce the vulnerability, any potential impact, and any suggestions for remediation if you have them.

Thank you for helping keep this project secure!
